为什么会有cookie

由于HTTP协议无状态这一特性，当客户端请求服务器时，每次都是一个新的请求，浏览器并不能记住请求记录。为了解决这个问题。产生了cookie。

cookie是什么？

cookie是由服务器下发给浏览器的一小段数据。

cookie的三个主要用途

会话管理

保存用户登录状态、保存用户游戏记录、购物车以及其他需要记录的信息

个性化设置

用户偏好、用户自定义设置、主题等

浏览器行为跟踪

如跟踪分析用户行为等

并且，在继续发起相同二级域名请求（任何请求）时，浏览器必须附上这个cookie。

总结

  1. cookie是服务器下发给浏览器的一段字符串；
  2. 浏览器必须保存这个cookie（除非用户删除），存放在浏览器的localStorage；
  3. 之后发起相同二级域名请求（任何请求）时，浏览器必须附上这个cookie。

为什么会有session？

仅仅使用cookie用来保存数据在浏览器上有以下几个弊端：

  1. cookie的存储量有限，上限是4k;；
  2. cookie将用户信息存储在浏览器，容易被获取，安全性较低；
  3. cookie全部存在浏览器，服务端无法验证，这样伪造和仿冒会更加容易。

因此我们需要通过**session**将用户数据隐藏在服务器。

session是什么？

session 代表着服务器和客户端一次会话的过程。Session 对象存储特定用户会话所需的属性及配置信息。这样，当用户在应用程序的 Web 页之间跳转时，存储在 Session 对象中的变量将不会丢失，而是在整个用户会话中一直存在下去。当客户端关闭会话，或者 Session 超时失效时会话结束。

cookie配合session

  1. 客户端请求服务器时，服务器创建一个**session**会话（存放了用户会话数据），并生成一个唯一sessionID发给浏览器；
  2. 浏览器接收到服务器返回的 sessionID 信息后，会将此信息存入到 cookie 中，同时cookie记录此 sessionID 属于哪个域名；
  3. 当用户第二次访问服务器的时候，请求会自动判断此域名下是否存在 cookie 信息，如果存在自动将

cookie 信息也发送给服务端，服务端会从 cookie 中获取 sessionID，再根据 sessionID 查找对应的 session 信息，如果没有找到说明用户没有登录或者登录失效，如果找到 session 证明用户已经登录可执行后面操作。​

总结：cookie和session之间通过sessionID建立联系。​

        response.statusCode = 200;
        // 用户名密码匹配，set-cookie标记用户已登录

        session[random] = {user_id:user.id}
        response.setHeader("Set-Cookie", `session_id=${random};HttpOnly`);
        response.end();
      }
    });
  }

拓展1：禁用cookie

如果浏览器中禁止了 cookie，如何实现功能？

请求地址携带sessionID作为查询参数

每次请求中都携带一个 SessionID 的参数，也可以 Post 的方式提交，也可以在请求的地址后面拼接xxx?SessionID=123456…。

使用token

Token 机制。Token 机制多用于 App 客户端和服务器交互的模式，也可以用于 Web 端做用户状态管理。

拓展2：分布式session

如何考虑分布式 Session 问题？ 在互联网公司为了可以支撑更大的流量，后端往往需要多台服务器共同来支撑前端用户请求，那如果用户在 A 服务器登录了，第二次请求跑到服务 B 就会出现登录失效问题。

共享session

共享 Session，服务端无状态话，将用户的 Session 等信息使用缓存中间件来统一管理，保障分发到每一个服务器的响应结果都一致。​

cookie与session对比

  1. 作用范围不同： Cookie将数据存在客户端的内存和磁盘上；Session将值存在服务器端的内存中；

  2. 隐私策略不同： Cookie 存储在客户端，比较容易遭到不法获取，早期有人将用户的登录名和密码存储在 Cookie 中导致信息被窃取；Session 存储在服务端，安全性相对 Cookie 要好一些。

  3. 存值类型不同： Cookie存的值是String类型；session存的值是Object类型。

  4. 有效期不同： Cookie 可设置为长时间保持，比如我们经常使用的默认登录功能，Session 一般失效时间较短，客户端关闭或者 Session 超时都会失效。

  5. 存储大小不同： cookie对同一个网站最多只能存20个cookie，每个Cookie最大只能4KB；Session对存储数据的长度无要求；
