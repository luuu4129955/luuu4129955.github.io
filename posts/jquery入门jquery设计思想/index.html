前言

jQuery于2006 年发布。

  • 是目前寿命最长的库。
  • 是目前使用最广泛的JavaScript函数库。全球超 80%的网站在使用。
  • jQuery 的语法设计使得许多操作变得容易，如操作文档对象（document）、选择文档对象模型（DOM）元素、创建动画效果、处理事件、以及开发 Ajax 程序。

设计思想一：选+操

jQuery的基本设计思想和主要用法，就是选择某个网页元素，然后对其进行某种操作。这是它区别于其他 Javascript 库的根本特点。

① 提供一个全局函数 window.jQuery()

嫌 window.jQuery()太长，给它取个别名！😉

window.jQuery = window.$ = jQuery; //jQuery简写为$

②jQuery 是构造函数

  • 是；
  • 因为 jQuery 确实能创建一个对象（但不需要用 new 方法创建）
  • jQuery 不是常规意义的构造函数

③ 该函数获取元素，并返回 jQuery 对象

用$(选择表达式)来获取网页元素：选择表达式可以是：

  1. CSS 选择器
  2. jQuery 特有表达式

一般地，这个jQuery对象就代指jQuery函数构造出来的对象，并且这个对象可以操作jQuery获取到的元素。例如：

  • Object是个函数，而Object对象是由Object函数构造的
  • Array是个函数，而Array对象/数组对象是由Array函数构造的
  • Function是个函数，而Function对象/函数对象是由Function函数构造的

④$开头的变量默认为 jQuery 对象

const div = $("div#test");
//会误以为div是一个DOM
//实际上div是jQuery构造的api对象

//以$开头命名变量，默认是jQuery对象
const $div = $("div#test");

设计思想二：闭包

闭包简单来说就是：一个函数用到了函数外部的变量。那么这个外部变量+这个函数就是一个闭包。闭包 MDN上面说到了 jQuery 就是选择元素，再操作它。jQuery()中封装了许多方法，都需要操作获取到的元素：所以把元素放到函数外部，然后在函数中调用它。​

设计思想三：链式操作

jQuery 设计思想之一就是链式操作：就是最终选中网页元素以后，可以对它进行一系列操作，并且所有操作可以连接在一起，以链条的形式写出来。例如：

$("div"); //获取div元素，并返回一个能操作div的函数
$("div").find("h3"); //获取div元素——→在div元素中找到h3标签
$("div").find("h3").html("Hello"); //获取div元素——→在div元素中找到h3标签——→h3内容改完Hello

这是 jQuery 最令人称道、最方便的特点。它的原理在于每一步的 jQuery 操作，返回的都是一个 jQuery 对象，所以不同操作可以连在一起，形成链式操作。

jQuery.js 好懂的写法

实现：获取这个类名为.test的 div，再给这个 div 添加一个新类名。

window.jQuery = function (selector) {
  const elements = document.querySelectorAll(selector); // 获取 selector 的全部元素（得到一个数组）
  // return elements 就直接返回这个通过选择器找到的元素。
  // 但jQuery却是：获取到元素后，没有返回这个元素，而是返回了可以操作这个元素的 api
  // 如下：
  // api 可以操作 elements
  // api 是个对象，里面包含各种可以操作 elements 的函数。
  //比如addClass()：给元素添加类名
  const api = {
    // 函数内访问了函数外部的变量，这就是「闭包」
    addClass(className) {
      // elements 是 addClass 这个函数外部的变量
      for (let i = 0; i < elements.length; i++) {
        // 遍历所有获取到的元素，添加类名
        elements[i].classList.add(className);
      }
      // return null
      return api;
      /*
       *方法仍返回 api，这个api是一个对象，里面又包含很多方法，
       *可通过返回值继续调用.addClass形成一个链条。
       *这就是链式风格
       */
    },
  };
  return api;
};

jQuery.js 直接 return api（this）

上面是：先声明一个 api 的方法，再 return 这个 api。

window.jQuery = function (selector) {
  const elements = document.querySelectorAll(selector)  // 获取 selector 的全部元素（得到一个数组）
  //闭包：函数调用外部变量
  //上面是定义一个api，再返回一个api
  //下面可以直接return这个api
  return {
    addClass(className) {
      for (let i = 0; i < elements.length; i++) {
        elements[i].classList.add(className);
      }
      return this;//this就是这个api
    },

}

jQuery 的常用 API

① 创建元素

创建新元素的方法非常简单，只要把新元素直接传入 jQuery 的构造函数就行了：

JS 重载

上面说到 jQuery 获取网页元素，其实也可以获取标签

  • jQuery 可以接受不同的参数（选择器或标签）
  • 用if判断参数类型
  • 根据参数类型执行语句
  • 代码示例

这种根据参数类型不同（或参数个数不同，封装 DOM APIs 用过）执行不同的语句的模式就是重载。

$("<p>Hello</p>");
$('<li class="new">new list item</li>');

② 删除元素

  1. 删除元素使用.remove()和.detach()。

两者的区别在于：

  • 前者不保留被删除元素的事件
  • 后者保留被删除元素的事件，有利于重新插入文档时使用。
  1. 清空元素内容（但是不删除该元素）使用.empty()。

③ 移动元素

两种方法：来操作元素在网页中的位置移动。

  1. 直接移动该元素
  2. 移动其他元素，使得目标元素达到我们想要的位置

例如：div 元素，p 元素，现在要让 div 元素在 p 元素后面：

$("div").insertAfter($("p")); //方法1：使用.insertAfter()，把div元素移动p元素后面
$("p").after($("div")); //方法2：使用.after()，把p元素加到div元素前面

两种方法实现效果相同，但返回的元素不一样：方法 1 返回 div 元素，方法 2 返回 p 元素。使用这种模式的操作方法，一共有四对：

.insertAfter()和.after() //在现存元素的外部，从后面插入元素
.insertBefore()和.before() //在现存元素的外部，从前面插入元素
.appendTo()和.append() //在现存元素的内部，从后面插入元素
.prependTo()和.prepend() //在现存元素的内部，从前面插

④ 读写元素（getter/setter）

使用同一个函数来实现读写（根据参数个数，判断读还是写，即重载）

$("h1").html(); //html()没有参数，表示取出h1的值
$("h1").html("Hello"); //html()有参数Hello，表示对h1进行赋值

$div.text(?) // 读写文本内容  // 传了参数就是「写」，不传参数就是「读」
$div.html(?) // 读写HTML内容  // 传了参数就是「写」，不传参数就是「读」
$div.attr('title', ?）  // 读写属性
$div.css({color: 'red'})  // 读写style // 注意方法名是css
$div.on('click', fn)
$div.off('click', fn)
