组件（推荐函数组件）

  • React 元素就是由React.createElement()API 创建的一个虚拟DOM对象
  • 在 React 中，一个返回 React 元素的函数可以当作一个组件

React.createElement 逻辑

  1. 传的字符串’div'：就新建一个 div 标签
  2. 传的函数：就调用函数，并获取其返回值
  3. 传的类：就在类的前面加 new关键字（会执行constructor）。获取一个组件对象，然后调用对象的render方法，获取其返回值。

1.函数组件

怎么写一个函数组件：

function Welcome(props) {
  return <h1>Hello,{props.name}</h1>;
}

使用这个组件：

<Welcome name="crystal" />

2.类组件

怎么写一个类组件：

class Welcome extends React.Component {
  render() {
    return <h1>Hello,{this.props.name}</h1>;
  }
}

使用这个组件：

<Welcome name="crystal" />

总结：会被babel翻译为React.createElement(Welcome)​

添加 props（外部数据）

关注如何读取这个 props，props 不能写

类组件

爸爸给儿子传值=儿子读爸爸的值直接读取属性：this.props.xxx

函数组件

儿子给孙子传值=孙子读儿子的值直接读取参数：props.xxx​

添加 state（内部数据）

关注state的初始化，读，写

类组件

初始化 state

constructor() {
    super();
    this.state = {
      n: 0
    };
  }

读 state

this.state.n;

写 state

this.state是异步执行的，不会立即改变n的值

//this.state

//1.内部直接返回一个新对象
this.state({ n: this.state.n + 1 });

//推荐写法2.内部写一个函数，接受一个旧的state参数
this.setState((state) => {
  const n = state.n + 1;
  return { n };
});

函数组件

useState返回数组，第一项读，第二项写

初始化 state

const [n, setN] = React.useState(0); //0就是初始化的值

读 state

n就是读到的值：

{
  n;
}

写 state

setN(n+1)永远不会改变n的值，它会产生新的n。

<button onClick={() => setN(n + 1)}>+1</button>

复杂 state

如果 state 里面有 n,m2 个属性

类组件

会自动合并第一层的 2 个属性，this.setState改变 n 的值，m 还是会沿用初始值，并不会被覆盖为undefined​

但是不会合并第二层属性自己使用Object.assign或者...操作符把不变属性拷过来

函数组件

不会自动合并如果 state 里面有多个属性，并且使用下面这种写法

const [state, setState] = React.useState({ n: 0, m: 0 });

这种写法 ，

  • 由于只改变一个值时，另一个值会被覆盖为undefined
  • 所以每次setState写值时，需要将其余不变属性拷贝一份
<button onClick={() => setState({ ...state, n: state.n + 1 })}>+1</button>

结论：

永远不要依赖自动合并，学会使用...或者Object.assign()

事件绑定

onClick、onKeyPress

类组件

//addN() 写在组件里，是一个普通函数
class Son extends React.Component{
  constructor() {
    super();
    this.state = {
      n: 0
    };
  }

  addN() {
      this.setState((state) => {
        const n = state.n + 1;
        return { n };
      });
    }

  //render里的事件绑定这么写
  render() {
    return (
        <button onClick={() => this.addN()}>+1</button>
    );
  }
}

//1.传一个函数给onClick即可。注意C大写
<button onClick={() => this.addN()}>n+1</button>

//2.错误写法
<button onClick={this.addN}>n+1</button>
//这样写，会让this.addN里的this指向window
//React 是这么调用的：button.onClick.call(null,event)。这里的null就是this。this为空则指向window

//3.可行写法，但太长
//返回了一个绑定了当前this的新函数
<button onClick={this.addN.bind(this)}>n+1</button>

改良版：

//addN 写在constructor里,是一个箭头函数
class Son extends React.Component {
  constructor() {
    uper();
    this.state = {
      n: 0,
    };

    this.addN = () =>
      this.setState((state) => {
        const n = state.n + 1;
        return { n };
      });
  }

  //render里的事件绑定这么写
  render() {
    return <button onClick={this.addN}>+1</button>;
  }
}

​

最终最优雅写法：

//addN 写在组件里，是一个箭头函数
//直接写在组件里，比改良版少写了一个this
class Son extends React.Component {
  constructor() {
    super();
    this.state = {
      n: 0,
    };
  }

  addN = () =>
    this.setState((state) => {
      const n = state.n + 1;
      return { n };
    });

  //render里的事件绑定这么写
  render() {
    return <button onClick={this.addN}>+1</button>;
  }
}
