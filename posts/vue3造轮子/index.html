添加事件&属性绑定

给组件添加 click、focus、mouseover 事件

父组件 Button，子组件 ButtonD

  • 父组件引用子组件模板
  • 直接在引用的子组件模板上添加事件
  • 那么，这些添加的事件会直接绑定到子组件的根元素上

注 1：如果只希望子组件内部某一元素（如 button 标签）绑定事件，看下面步骤：

  1. 在子组件中，将继承属性的值改为 false（这样事件不会绑定在子组件的任何一个元素上）
inheritAttrs=false
  1. 将事件绑定在子组件的 button 标签上 （$attrs）

$attrs是尤雨溪发明的，它默认包含了所有的属性，以对象的形式来展现这些属性。然后使用v-bind="$attrs"来绑定（即$attrs这个对象上的所有属性都通过v-bind绑定到button标签上）

<template>
  <div>
    <button v-bind="$attrs">
      <slot />
    </button>
  </div>
</template>

注 2：如果**$attrs**的属性想分开绑定到不同的元素上：

  1. 使用setup()的context参数获取到$attrs
  2. size和其他属性...rest切割开，分别绑定到不同元素上

    es6 剩余操作符 rest ES6 引入 rest 参数（形式为…变量名），用于获取函数的多余参数，这样就不需要使用 arguments 对象了。rest 参数为形参，如上面例子中的 theArgs，是一个接收函数调用时指向实参的参数变量，rest 参数搭配的变量是一个数组，该变量将多余的参数放入数组中，rest 操作符将多余参数变为数组：

<template>
  <div :size="size">
    <button v-bind="rest">
      <slot />
    </button>
  </div>
</template>

<script lang="ts">
export default {
  setup(props, context) {
    const { size, ...rest } = context.attrs;
    return { size, rest };
  },
};
</script>

支持 visible 属性

思路：

  1. 子组件 props 中声明 visible 属性，默认值为 false
  2. 父组件声明一个变量 x，（x 是要传给子组件的）
  3. 在父组件引入的子组件模板中，绑定 visible 属性，值是 x
  4. 子组件添加 v-if 指令，visible 为 true 就显示，false 就隐藏
  5. 父组件添加 button 按钮，给 button 添加一个点击事件，用户点击则将 visible 的值取反

支持点击关闭

上面 visible 属性的值，在子组件中是不能改的，只能由父组件传过来。有 4 个地方需要实现点击关闭：一. 点“×”关闭**思路：**不能改变子组件 props 的属性值，可以通过 setup()的第二个参数 context 结合父组件来实现

  1. 子组件的“×”元素上添加 close 事件，然后使用 context.emit()触发事件
<script lang="ts">
import ButtonD from "./ButtonD.vue";
export default {
  components: { ButtonD },
  props: {
    visible: {
      type: Boolean,
      default: false,
    },
  },
  setup(props, context) {
    const close = () => {
      context.emit("update:visible", false);
    };
    return { close };
  },
};
</script>

context.emit("事件名",事件参数)
事件名默认写法：update:props里声明的属性名(即你要更新的属性)
  1. 在父组件引用的子组件模板中，监听用户事件并把更新的属性值传给子组件（使用 Vue3：v-model）
<DialogD v-model:visible="x"></DialogD>

二. 点遮罩层关闭**需求：**可以给遮罩层元素添加与“×”按钮相同的 close 事件。**但是为了防止用户误点遮罩层，导致对话框关闭。**需要加一个属性：closeOnClickOverlay（类型为布尔，默认值为 true）​

**思路：**添加一个 closeOnClickOverlay 属性。添加一个点击事件 onClickOverlay，如果属性值为 true，则调用 close()。​

三 . 点击 OK，cancel 实现关闭需求：点击 OK 时，需要判断内容是否为空，为空则不能关闭思路：

  • 子组件定义 OK，cancel 属性
  • 子组件元素添加点击事件，分别绑定 OK，cancel 函数
  • 父组件给 OK，cancel 分别绑定 f1()，f2()函数

​

四. dialog 支持 title 和 content**思路：**使用插槽 slot具名插槽（给插槽一个名字）用法：

  • 子组件给插槽命名
  • 父组件用 template 标签包住，里面写 v-slot:插槽名

五. 新组建 teleport由于 CSS 的层级上下文，可能 dialog 会被遮挡。**解决办法：**使用内置组件 teleport ，组件有个 to 属性，可以把组件内的元素移到 to 属性值的后面（即移出当前的层级上下文）六.一句话打开 dialog**思路：**给 show 按钮添加点击事件，并绑定一个 showDialog 函数，一旦点击，就执行 openDialog 函数openDialog 函数（动态挂载）

import DialogD from "../lib/DialogD.vue";
import { createApp, h } from "vue";
export const openDialog = (options) => {
  const { title, content, ok, cancel, closeOnClickOverlay } = options;
  const div = document.createElement("div");
  document.body.appendChild(div);
  const close = () => {
    app.unmount(div);
    div.remove();
  };
  const app = createApp({
    render() {
      return h(
        DialogD,
        {
          visible: true,
          "onUpdate:visible": (newVisible) => {
            if (newVisible === false) {
              close();
            }
          },
          ok,
          cancel,
          closeOnClickOverlay,
        },
        { title, content }
      );
    },
  });
  app.mount(div);
};

  1. 如何在运行时检查子组件的类型（防御性编程）

如果开发者不是使用的 Tab 组件，那么就报一个错

Vue 基本原理：组件写好被导出为一个对象，对象里面有 render()函数

TabsD.vue

<template>
  <div>
    TabsD
    <component :is="defaults[0]" />
    <component :is="defaults[1]" />
  </div>
</template>

<script lang="ts">
  import TabD from "./TabD.vue";
  export default {
    setup(props, context) {
      const defaults = context.slots.default();
      defaults.forEach((tag) => {
        if (tag.type !== TabD) {
          throw new Error("TabsD 子标签必须是 TabD ");
        }
      });
      return {
        defaults,
      };
    },
  };
</script>
